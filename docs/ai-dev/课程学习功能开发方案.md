# 课程学习功能开发方案

## 一、需求概述

在前端实现课程学习功能，用户可对课程中的单词进行逐字拼写答题。

### 核心功能

- 后端返回10个单词，前端遍历对这10个单词进行拆字母拼写答题
- 用户每答对一个单词进入下一个单词答题
- 所有单词答完后可继续加载下一批10个单词

### 页面元素

| 元素 | 类型 | 说明 |
|------|------|------|
| 课程标题 | 文本 | 从 URL query 参数获取 |
| 单词 | 文本 | 紫色大字，可模糊显示 |
| 音标按钮 | 按钮 | 点击朗读单词（调用 useAudio） |
| 模糊切换按钮 | 按钮 | 切换单词显示/隐藏（默认开启） |
| 进度文本 | 文本 | "第 1 / 10 个" |
| 释义 | 文本 | 灰色卡片，为 null 可省略 |
| 翻译 | 文本 | 灰色卡片 |
| 拼写输入框 | 输入框 | 逐字输入，实时校验边框颜色 |
| 上一个/下一个按钮 | 按钮 | 切换单词，下一个需全部正确才可点击 |
| 完成模态框 | 弹窗 | 学完10个单词后弹出，选择继续或返回 |

## 三、API 层设计

**文件**: `apps/web/src/api/server/learn.ts`

- `getLearnWordList(courseId)` - 调用 `GET /learn/:courseId`，获取最多10个未学单词
- `markWordsMastered(wordIds)` - 调用 `POST /learn`，批量标记单词已掌握，返回用户总单词数

## 四、学习 Hook 设计

**文件**: `apps/web/src/composables/business/learn/useLearn.ts`

### 状态

| 状态 | 类型 | 说明 |
|------|------|------|
| `words` | `Ref<Word[]>` | 当前单词列表（最多10个） |
| `currentIndex` | `Ref<number>` | 当前单词索引 |
| `currentWord` | `ComputedRef` | 当前单词对象 |
| `isLoading` | `Ref<boolean>` | 加载状态 |
| `isBlur` | `Ref<boolean>` | 模糊状态（默认开启） |
| `masteredIds` | `Ref<string[]>` | 本次已掌握的单词ID |
| `showGroupCompleteModal` | `Ref<boolean>` | 控制完成模态框显示 |

### 计算属性

| 属性 | 说明 |
|------|------|
| `canGoNext` | 是否可切换下一个 |
| `canGoPrev` | 是否可切换上一个 |
| `isAllCompleted` | 是否全部完成 |

### 方法

| 方法 | 说明 |
|------|------|
| `fetchWords(courseId)` | 获取单词列表，失败时跳转 404 |
| `nextWord()` | 下一个单词 |
| `prevWord()` | 上一个单词 |
| `toggleBlur()` | 切换模糊状态 |
| `markCurrentMastered()` | 标记当前单词已掌握 |
| `submitMastered(courseId)` | 提交已掌握单词，更新 wordNumber，显示模态框 |
| `continueLearning(courseId)` | 关闭模态框，加载下一组单词 |
| `playAudio(word)` | 播放单词发音 |

## 五、页面组件设计

**文件**: `apps/web/src/views/courses/learn/[courseId].vue`

### 路由

- 路由路径: `/courses/learn/:courseId`（文件路由自动生成）
- Query 参数: `title` (课程标题)
- 需要登录: 是

### 核心逻辑

1. **路由参数**: 从 `route.params.courseId` 获取课程ID，`route.query.title` 获取标题
2. **字符拆分**: 将当前单词拆分为字符数组，每个字符一个输入框
3. **输入校验**: 实时比较用户输入与目标字符，动态设置边框颜色
4. **焦点管理**: 输入正确自动跳转下一个输入框，退格自动跳转上一个；切换单词时自动聚焦第一个输入框
5. **单词切换**: 点击"下一个"时标记当前单词已掌握，切换到下一个或显示完成模态框
6. **模态框交互**: 10个单词完成后弹出模态框，"下次吧"返回课程列表，"再来一组"继续学习
7. **用户单词数更新**: 提交后将响应体的 `data` 字段覆盖到 `userStore.user.wordNumber`

### 样式规范

| 元素 | 样式 |
|------|------|
| 单词显示 | `text-4xl font-bold text-purple-600` |
| 模糊效果 | `blur-sm` |
| 信息卡片 | `bg-gray-100 dark:bg-gray-800 rounded-lg p-4` |
| 输入框 | `w-10 h-12 text-center text-lg font-mono border-2 rounded` |
| 正确边框 | `border-green-500 bg-green-50` |
| 错误边框 | `border-red-500 bg-red-50` |
| 未输入边框 | `border-gray-300 dark:border-gray-600` |

## 六、交互流程

```
页面加载 → GET /learn/:courseId
  ├─ 成功 → 获取10个单词 → 显示学习内容
  │    ↓
  │  显示第一个单词（默认模糊）
  │    ↓
  │  用户逐字输入 → 实时校验边框颜色
  │    ├─ 正确 → 绿色边框 + 自动跳转下一个输入框
  │    ├─ 错误 → 红色边框
  │    └─ 未输入 → 灰色边框
  │    ↓
  │  全部正确 → "下一个"按钮启用
  │    ↓
  │  点击"下一个"
  │    ├─ 未完成10个 → 切换到下一个单词（自动聚焦第一个输入框）
  │    └─ 已完成10个 → POST /learn 提交 → 更新 userStore.wordNumber → 显示模态框
  │         ├─ 点击"下次吧" → 关闭模态框 → 跳转 /courses
  │         └─ 点击"再来一组" → 关闭模态框 → GET /learn/:courseId 加载下一组
  │
  └─ 失败 → 跳转 /404（复用 [...slug].vue 组件）
```

---

## 七、注意事项

1. **路由**: 使用文件路由自动生成，无需手动配置
2. **登录守卫**: 通过 `definePage({ meta: { requiresAuth: true } })` 声明
3. **暗色模式**: 所有颜色需适配 `dark:` 变体
4. **响应式**: 使用 Tailwind 响应式类，适配移动端
5. **大小写不敏感**: 拼写校验时忽略大小写
6. **退格处理**: 当前输入框为空时，退格自动聚焦上一个输入框
7. **焦点管理**: 使用 `nextTick` 确保 DOM 更新后再聚焦输入框
8. **404 路由**: 在 `router/index.ts` 中添加 `/404` 命名路由，复用 `[...slug].vue` 组件，课程不存在或请求失败时跳转

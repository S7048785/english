# 支付订单过期定时任务实现文档

## 概述

实现了一个定时任务，每15分钟扫描数据库中的`PaymentRecord`表，将超过15分钟未支付的订单状态更新为`TRADE_CLOSED`。

## 实现步骤

### 1. 安装依赖

```bash
cd server
pnpm add @nestjs/schedule dayjs
```

### 2. 配置ScheduleModule

**文件**: `server/apps/server/src/app.module.ts`

```typescript

@Module({

	// ...
```

### 3. 创建PaymentExpirationService

**文件**: `server/apps/server/src/pay/payment-expiration.service.ts`

```typescript
import {Injectable, Logger} from '@nestjs/common';
import {Cron} from '@nestjs/schedule';
import {PrismaService} from '@libs/shared';
import {TradeStatus} from '@libs/shared/generated/prisma/enums';
import dayjs from 'dayjs';

@Injectable()
export class PaymentExpirationService {
	private readonly logger = new Logger(PaymentExpirationService.name);

	constructor(private readonly prisma: PrismaService) {
	}

	@Cron('*/15 * * * *') // 每15分钟执行
	async handleExpiredPayments() {
		const fifteenMinutesAgo = dayjs().subtract(15, 'minute').toDate();

		try {
			const expiredOrders = await this.prisma.paymentRecord.updateMany({
				where: {
					tradeStatus: {
						in: [TradeStatus.NOT_PAY, TradeStatus.WAIT_BUYER_PAY]
					},
					createdAt: {lt: fifteenMinutesAgo}
				},
				data: {
					tradeStatus: TradeStatus.TRADE_CLOSED
				}
			});

			if (expiredOrders.count > 0) {
				this.logger.log(`已关闭 ${expiredOrders.count} 个过期订单`);
			}
		} catch (error) {
			this.logger.error('处理过期订单失败', error);
		}
	}
}
```

### 4. 注册服务到PayModule

**文件**: `server/apps/server/src/pay/pay.module.ts`

```typescript
import {Module} from '@nestjs/common';
import {PayService} from './pay.service';
import {PayController} from './pay.controller';
import {PaymentExpirationService} from './payment-expiration.service';

@Module({
	controllers: [PayController],
	providers: [PayService, PaymentExpirationService],
})
export class PayModule {
}
```

## 关键说明

### Cron表达式

- `*/15 * * * *` 表示每15分钟执行一次
- 执行时间点：0分、15分、30分、45分

### 过期判断逻辑

- 查询条件：`tradeStatus` 为 `NOT_PAY` 或 `WAIT_BUYER_PAY`
- 时间条件：`createdAt < 当前时间 - 5分钟`
- 更新操作：将 `tradeStatus` 更新为 `TRADE_CLOSED`

### 日志记录

- 使用NestJS内置的Logger
- 只在有过期订单时记录日志，避免日志刷屏
- 错误时记录错误日志

## 测试方法

```bash
pnpm server
```

2. 观察控制台日志，应看到类似输出（如果有过期订单）：

```
[Nest] 12345  - 2026/06/06 10:30:00   LOG [PaymentExpirationService] 已关闭 3 个过期订单
```

## 注意事项

1. **依赖关系**：确保`dayjs`和`@nestjs/schedule`已正确安装
2. **数据库索引**：如果表数据量大，建议在`createdAt`和`tradeStatus`字段上添加复合索引
3. **分布式部署**：如果多实例部署，需要考虑分布式锁避免重复处理
4. **时区处理**：`dayjs`默认使用本地时区，确保服务器时区配置正确
